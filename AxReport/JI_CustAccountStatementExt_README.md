# JI_CustAccountStatementExt — SSRS Report Design

## Overview

This report is a **duplicate** of the standard `CustAccountStatementExt` report,
extended with three additional Reporting Currency columns.

## Creation Steps (Visual Studio)

1. **Duplicate the standard report**
   - In AOT, navigate to: Reports > CustAccountStatementExt
   - Right-click > Duplicate in model > Select model: `JI_CustAccountStatementRC`
   - Rename to: `JI_CustAccountStatementExt`

2. **Refresh the dataset**
   - Open the duplicated report in the SSRS designer
   - Right-click the dataset bound to `CustAccountStatementExtTmp` > Refresh
   - Verify the four new fields appear:
     - `JI_ReportingCurrencyDebit`
     - `JI_ReportingCurrencyCredit`
     - `JI_ReportingCurrencyBalance`
     - `JI_ReportingCurrencyCode`

3. **Add columns to the transaction detail band (tablix)**
   - Position: To the **right** of the existing Balance column
   - Add three new columns:

   | Column | Field | Header Expression | Format |
   |--------|-------|-------------------|--------|
   | Debit (RC) | `JI_ReportingCurrencyDebit` | `="Debit (" & Fields!JI_ReportingCurrencyCode.Value & ")"` | Same as existing Debit column |
   | Credit (RC) | `JI_ReportingCurrencyCredit` | `="Credit (" & Fields!JI_ReportingCurrencyCode.Value & ")"` | Same as existing Credit column |
   | Balance (RC) | `JI_ReportingCurrencyBalance` | `="Balance (" & Fields!JI_ReportingCurrencyCode.Value & ")"` | Same as existing Balance column |

4. **Formatting**
   - Match the numeric format of the existing Debit/Credit/Balance columns
     (typically: `#,##0.00;(#,##0.00);-`)
   - Match font, alignment, and border styles
   - Verify the report fits on A4/Letter — may need to adjust column widths
     or switch to landscape orientation

5. **Deploy**
   - Right-click the report in Solution Explorer > Deploy Report
   - Verify in the browser: AR > Inquiries and Reports > Customer > Customer account statement

## Design Name

- Report object: `JI_CustAccountStatementExt`
- Design name: `Report` (the precision design)
- Full SSRS reference: `JI_CustAccountStatementExt.Report`

## Notes

- The `.rdl` file is a binary/XML artefact generated by Visual Studio SSRS designer.
  It cannot be authored as a text file and must be created within the D365 F&O
  development environment.
- After creation, commit the `.rdl` file to this repository for version control.
